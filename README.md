API REST desarrollada con **ASP.NET Core** para la gestión de usuarios y bebidas, aplicando **Clean Architecture**, principios **SOLID** y buenas prácticas de desarrollo backend.

El objetivo principal del proyecto fue profundizar conocimientos en arquitectura de software, diseño de dominio, pruebas unitarias, contenedorización y despliegue en la nube.

---

## 🛠 Tecnologias

- .NET 8
- ASP.NET Core Web API
- C#
- Entity Framework Core 8
- SQL Server
- AutoMapper
- Clean Architecture
- SOLID Principles
- Value Objects
- Dependency Injection
- Global Exception Handling Middleware
- Swagger / OpenAPI
- Docker
- Microsoft Azure
- xUnit
  
## 🧠 Conceptos aplicados

Durante el desarrollo se implementaron diferentes patrones y buenas prácticas de desarrollo backend.

- Clean Architecture
- Principios SOLID
- Value Objects
- Dependency Injection
- Entity Framework Core
- AutoMapper
- Middleware para manejo centralizado de errores
- Separación de responsabilidades
- Arquitectura desacoplada
---
## 🗄 Base de datos

La aplicación utiliza **SQL Server** como motor de base de datos mediante **Entity Framework Core**.

La base de datos se encuentra desplegada en **Microsoft Azure**.

---

## 🐳 Docker

La API fue contenedorizada mediante un **Dockerfile**, permitiendo su ejecución en cualquier entorno compatible con Docker y facilitando su despliegue en la nube.

---

## ☁ Despliegue

La aplicación fue desplegada en **Microsoft Azure** utilizando un contenedor Docker.

---

## 🧪 Testing

El proyecto incluye pruebas unitarias desarrolladas con **xUnit**.

Se implementaron pruebas para las capas:

- Domain
- Application

con el objetivo de validar la lógica de negocio y el correcto funcionamiento de los casos de uso.
